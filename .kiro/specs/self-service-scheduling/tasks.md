# Implementation Plan: Self-Service Scheduling

## Overview

This plan implements the self-service scheduling feature for the Jobuler platform, enabling groups to let members request shifts from available slots. The implementation follows the existing Clean Architecture layering (Api → Application → Domain ← Infrastructure) and uses PostgreSQL advisory locks for concurrency-safe slot claiming. Tasks are ordered to build foundational domain entities first, then services, then controllers, and finally background jobs and notifications.

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP
- Each task references specific requirements for traceability
- Checkpoints ensure incremental validation
- Property tests validate universal correctness properties from the design document (Properties 1–37)
- Unit tests validate specific examples and edge cases
- The implementation uses C# with the existing .NET stack (EF Core, MediatR, Hangfire, FluentValidation, FsCheck)
- All new tables follow existing RLS patterns for tenant isolation
- Advisory locks are transaction-scoped — no explicit release needed
