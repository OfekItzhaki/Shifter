# 570 — Self-Service Scheduling EF Core Migration

## Phase

Self-Service Scheduling — Infrastructure Layer

## Purpose

Generates the EF Core migration that creates all new database tables, columns, indexes, and RLS policies required for the self-service scheduling feature. This migration transforms the domain model and EF configurations into actual PostgreSQL schema changes.

## What was built

| File | Description |
|------|-------------|
| `apps/api/Jobuler.Application/Persistence/Migrations/20260525235005_AddSelfServiceScheduling.cs` | Migration adding all self-service scheduling tables, indexes, and RLS policies |
| `apps/api/Jobuler.Application/Persistence/Migrations/20260525235005_AddSelfServiceScheduling.Designer.cs` | Auto-generated designer file for the migration |
| `apps/api/Jobuler.Application/Persistence/Migrations/AppDbContextModelSnapshot.cs` | Updated model snapshot reflecting the new schema |

## Key decisions

- **Single migration**: All 7 new tables and the `scheduling_mode` column on `groups` are created in one migration for atomic deployment.
- **RLS via raw SQL**: Following the existing project pattern (see `AddSpaceManagement` migration), RLS is enabled and `tenant_isolation` policies are created using `migrationBuilder.Sql()` calls.
- **Down method drops RLS first**: The rollback path drops RLS policies before dropping tables to avoid PostgreSQL errors.
- **Default values**: `scheduling_mode` defaults to `'AutoGenerated'` so existing groups are unaffected.
- **All design indexes included**: idx_scheduling_cycles_group_dates, idx_shift_templates_group, idx_shift_slots_cycle, idx_shift_slots_group_date, idx_shift_requests_person_cycle, idx_shift_requests_slot, idx_shift_requests_no_dup, idx_waitlist_no_dup, idx_waitlist_slot_position, idx_swap_requests_status.

## Tables created

| Table | Purpose |
|-------|---------|
| `self_service_configs` | Group-level configuration (min/max shifts, request window offsets, etc.) |
| `scheduling_cycles` | Scheduling periods with request window timestamps |
| `shift_templates` | Recurring weekly shift patterns |
| `shift_slots` | Concrete time slots generated from templates |
| `shift_requests` | Member requests to claim slots |
| `waitlist_entries` | Ordered queue for full slots |
| `swap_requests` | Shift swap proposals between members |

## How to run / verify

```bash
cd apps/api
dotnet build --no-restore
dotnet ef database update --project Jobuler.Application --startup-project Jobuler.Api --context AppDbContext
```

Build should succeed. Database update applies the migration to a running PostgreSQL instance.
