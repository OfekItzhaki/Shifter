# Step 623 — Pick Group Filter Property Test

## Phase

Feature: shift-picker-lite — Property-based testing

## Purpose

Validates that the `filterSelfServiceGroups` utility correctly filters groups by `schedulingMode === "SelfService"` across all possible input combinations. This property test ensures no groups with other scheduling modes leak through the filter, and that all self-service groups are preserved.

## What was built

| File | Description |
|------|-------------|
| `apps/web/__tests__/selfService/pickGroupFilter.property.test.ts` | Property-based test using fast-check that generates arbitrary arrays of groups with random scheduling modes and asserts the filter returns exactly the self-service subset |

## Key decisions

- Used `fc.oneof` to generate mixed scheduling modes including "SelfService", "AutoGenerated", undefined, and arbitrary strings — covering the full input space
- Minimum 100 iterations as specified in the design document
- Three assertions per run: all results are SelfService, no non-SelfService groups in output, all SelfService inputs appear in output

## How it connects

- Tests the `filterSelfServiceGroups` function from `apps/web/lib/utils/pickGroupFilter.ts` (created in step 620)
- Validates Requirement 2.1 (self-service group filtering)
- Complements the unit tests in `pickGroupFilter.test.ts` with exhaustive property coverage

## How to run / verify

```bash
cd apps/web
npx vitest run __tests__/selfService/pickGroupFilter.property.test.ts
```
