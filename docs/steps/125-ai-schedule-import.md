# Step 125 — AI Schedule Import

## Phase
Phase 4 — AI Features

## Purpose
Provide a single-button import flow that accepts Excel files, images, or PDFs of schedules, uses AI (GPT-4o vision/text) to extract people, tasks, and assignments, shows a preview for admin review, and creates a draft schedule version on confirmation.

## Key decisions

1. **Dedicated ImportController** — Separated import endpoints from AiController for cleaner routing (`/import/parse` and `/import/confirm`).
2. **Two-step flow** — Parse returns a preview DTO; confirm only runs after admin review. AI never auto-applies changes.
3. **Draft version without direct assignments** — The confirm command creates people, tasks, and a draft `ScheduleVersion` with assignment metadata in `SummaryJson`. Direct assignment creation requires task slots which are normally generated by the solver.
4. **File validation** — Server-side validation of file extension (.xlsx, .xls, .csv, .png, .jpg, .jpeg, .pdf) and size (10MB max).
5. **No architecture violations** — Removed the `Infrastructure.AI.NoOpAiAssistant` type check from Application layer; instead checks for empty AI response.
6. **Fixed compile errors** — The old `SmartImportCommand.cs` referenced `GroupMembers` (doesn't exist) and `Infrastructure.AI` namespace (architecture violation). Both fixed.

## How it connects

- Uses existing `IAiAssistant.ParseScheduleFileAsync()` (implemented in `OpenAiAssistant` with GPT-4o vision)
- Uses existing `CreatePersonCommand` and `CreateGroupTaskCommand` for entity creation
- Creates `ScheduleVersion` drafts using the existing domain entity
- Frontend uses existing `apiClient` (axios) and `Modal` component
- Translations follow existing `next-intl` pattern with `useTranslations("import")`

### Manual testing
1. Log in as admin, navigate to a group → Settings tab
2. Click "ייבוא חכם" button
3. Upload an Excel file or image of a schedule
4. Review the parsed preview (people, tasks, assignments)
5. Toggle people on/off, then click "אשר וצור טיוטה"
6. Verify people and tasks are created, draft version exists
