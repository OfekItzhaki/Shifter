# 618 — Scheduling Mode Selector Wizard Step

## Phase
Self-Service Scheduling UI — Group Creation Wizard Extension

## Purpose
Extends the `CreateGroupWizard` from a single-step flow (name + template) to a multi-step wizard that includes scheduling mode selection. This allows space admins to choose between automatic scheduling and self-service scheduling when creating a group.

## What was built

| File | Description |
|------|-------------|
| `apps/web/components/groups/selfService/SchedulingModeSelector.tsx` | New component presenting two visual cards for scheduling mode selection with permanent warning |
| `apps/web/components/CreateGroupWizard.tsx` | Extended to 3-step wizard: name → scheduling mode → template (mode-conditional) |
| `apps/web/app/groups/page.tsx` | Updated `handleCreateWithWizard` to accept and apply `schedulingMode` via the PUT scheduling-mode endpoint |

## Key decisions

1. **Multi-step wizard pattern**: The wizard now has 3 steps (name → mode → template) with back/continue navigation, keeping each step focused.
2. **Mode-conditional template step**: When "AutoGenerated" is selected, the existing template picker (army-base, restaurant, etc.) is shown. When "SelfService" is selected, self-service-specific template options are shown.
3. **Two-call creation flow**: Group is created first (existing POST), then scheduling mode is set via PUT `/scheduling-mode` endpoint. This avoids modifying the backend create endpoint.
4. **Visual selection with radio-style indicators**: Each mode card has a circular selection indicator and distinct highlight styling when selected.
5. **Permanent warning**: An amber warning box is always visible in the mode selection step, informing the user that the choice cannot be changed.
6. **Continue button disabled until selection**: The continue button in step 2 is disabled until a mode is selected, preventing accidental progression.

## How it connects

- Uses existing `selfService.modeSelector.*` i18n keys (added in step 611)
- Calls the existing `PUT /spaces/{spaceId}/groups/{groupId}/scheduling-mode` backend endpoint (implemented in step 572)
- The `SchedulingMode` type is exported for use by the parent page and future components
- The `ModeWarningDialog` (task 14.2) will build on this by adding a confirmation dialog before final submission

## How to run / verify

1. Start the dev server: `cd apps/web && npm run dev`
2. Navigate to the Groups page
3. Click "Create Group"
4. Enter a group name and click Continue
5. Verify the scheduling mode selection step appears with two cards and a warning
6. Verify the continue button is disabled until a mode is selected
7. Select "סידור אוטומטי" → verify existing template picker appears
8. Go back, select "בחירת משמרות" → verify self-service template options appear
9. TypeScript check: `npx tsc --noEmit` (no errors in modified files)
