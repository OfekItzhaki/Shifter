# 566 — ShiftSlot Entity and ShiftSlotStatus Enum

## Phase

Self-Service Scheduling — Domain Layer (Task 1.5)

## Purpose

Defines the `ShiftSlot` domain entity and `ShiftSlotStatus` enum for the self-service scheduling feature. A ShiftSlot represents a concrete time slot generated from a ShiftTemplate for a specific date, tracking capacity and current fill count so members can request shifts.

## What was built

| File | Description |
|------|-------------|
| `apps/api/Jobuler.Domain/Scheduling/ShiftSlotStatus.cs` | Enum with `Open` and `Closed` values |
| `apps/api/Jobuler.Domain/Scheduling/ShiftSlot.cs` | Domain entity with all properties (SpaceId, GroupId, GroupTaskId, ShiftTemplateId, SchedulingCycleId, Date, StartTime, EndTime, Capacity, CurrentFillCount, Status), factory method, and mutation methods |

## Key decisions

- Placed in `Jobuler.Domain.Scheduling` namespace alongside existing scheduling entities (Assignment, ScheduleRun, etc.)
- Follows existing conventions: private setters, private parameterless constructor, static `Create` factory method
- Inherits from `AuditableEntity` (provides Id, CreatedAt, UpdatedAt, Touch()) and implements `ITenantScoped`
- `Create` factory initializes `CurrentFillCount = 0` and `Status = Open` per design spec (Requirement 3.2)
- Added `IncrementFillCount()`, `DecrementFillCount()`, `Close()`, `Reopen()`, and `HasAvailableCapacity()` methods for encapsulated state transitions
- `DecrementFillCount()` guards against going below zero

## How it connects

- Generated by `SlotGenerationService` from `ShiftTemplate` entities (Task 5.3)
- Referenced by `ShiftRequest` entities when members claim slots (Task 1.6)
- Referenced by `WaitlistEntry` entities for full-slot queuing (Task 1.7)
- Locked via `ISlotLockService` (advisory locks) during request processing (Task 2.3)
- EF Core configuration will be added in Task 2.1
