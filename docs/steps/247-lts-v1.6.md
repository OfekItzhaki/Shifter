# Step 247 — LTS v1.6: Solver Pipeline Restored, Home-Leave, Security Hardening

## Phase
Phase 10 — Production Stability (LTS Checkpoint)

## Purpose
This LTS marks the first fully working production deployment with the solver pipeline, home-leave scheduling, biometric login, color-coded roles, qualification templates, and multiple security/stability fixes.

## What changed since v1.4

### Solver Pipeline — Fixed and Optimized
- **Root cause**: VPS was on wrong branch (`feat/home-leave-scheduling`), solver container had old unoptimized code, timeout mismatch (CP-SAT 60s vs HTTP 20s), horizon off-by-one producing 0 slots
- **Constraint optimization**: Pre-computed timestamps in O(n²) loops — eliminated ~500K ISO string parses per solve
- **Horizon fix**: `AddDays(maxHorizon)` instead of `AddDays(maxHorizon - 1)` — was producing 0 slots with small horizons
- **Progress phases**: Solver worker reports `building_payload` → `solving` → `storing_results` to the frontend
- **Timing instrumentation**: Solver logs constraint build time vs CP-SAT solve time

### Home-Leave Scheduling
- Balance slider with base:home ratio display
- Per-person priority multiplier (0.5–3.0)
- Home-leave config panel hidden when closed base is OFF
- Preview endpoint (has known 422 issue with `trigger_mode: "preview"` — non-blocking)

### Security Hardening
- Solver port no longer exposed to internet (`expose` instead of `ports`)
- Postgres, Redis, MinIO bound to `127.0.0.1` only
- Token refresh mutex prevents concurrent refresh requests from revoking each other
- `[DisableRateLimiting]` on `/auth/refresh` endpoint
- Generic error messages — no DB internals exposed to clients
- Rate limit increased to 2000/min (500 was too low for SPA)

### Frontend Fixes
- Week navigation: `maxDate` changed from `solverHorizonDays` to 30 days ahead
- Horizon slider: min changed from 3 to 1 (allows 1-day planning)
- Platform page: hydration fallback + localStorage token check
- Join page: same hydration fix — checks localStorage directly
- 403 interceptor: no longer auto-redirects (pages handle their own 403)
- Draft GUID resolution: uses absolute indexing matching the normalizer

### Infrastructure
- DB migration: `schedule_runs.progress_phase` column (TEXT)
- `.env`: `SOLVER_TIMEOUT_SECONDS=60`, `SOLVE_TIMEOUT_SECONDS=90`
- HTTP client timeout: 90s (aligned with solver)
- Deploy workflow: `docker compose down --remove-orphans && docker compose up -d`

## Known Issues
- `trigger_mode: "preview"` rejected by solver Pydantic model (home-leave preview broken)
- Draft schedule shows different dates than published (GUID resolution partially fixed)
- Duplicating browser tab sometimes shows "not logged in" (Zustand hydration race)
- Push notifications fail with "A subject is required" (VAPID_SUBJECT not configured)

- Solver: 41/41 Python tests passing locally
- API: builds with 0 errors
- Frontend: TypeScript compiles with 0 errors
