# 568 — SchedulingMode Enum and Group Entity Update

## Phase
Self-Service Scheduling — Domain Layer Foundation

## Purpose
Introduces the `SchedulingMode` enum that determines how shifts are filled for a group — either via the solver (`AutoGenerated`) or via member requests (`SelfService`). This is the foundational toggle that gates all self-service scheduling behavior.

## What was built

| File | Description |
|------|-------------|
| `apps/api/Jobuler.Domain/Groups/SchedulingMode.cs` | New enum with `AutoGenerated` and `SelfService` values. |
| `apps/api/Jobuler.Domain/Groups/Group.cs` | Added `SchedulingMode` property (defaults to `AutoGenerated`) and `SetSchedulingMode()` setter method. |

## Key decisions

- **Placed in `Groups` namespace**: The enum is a property of the `Group` entity, following the same pattern as `GroupTemplateType` and `HomeLeaveMode`.
- **Default to `AutoGenerated`**: Existing groups continue to use solver-based scheduling. Self-service must be explicitly opted into (Requirement 1.4).
- **Setter method pattern**: Uses `SetSchedulingMode()` with `Touch()` call, consistent with all other Group property setters.
- **No validation in setter**: Mode change validation (checking for active requests) belongs in the Application layer command handler, not the domain entity. The domain entity is a simple state holder.

## How it connects

- Used by `TriggerSolverCommand` handler to short-circuit solver runs for SelfService groups (Requirement 1.2)
- Used by `ShiftRequestService` to reject requests for AutoGenerated groups (Requirement 1.3)
- Used by `ChangeSchedulingModeCommand` handler to toggle the mode with active-request guards (Requirement 1.5)
- The Infrastructure layer will map this enum to a `scheduling_mode` text column on the `groups` table
